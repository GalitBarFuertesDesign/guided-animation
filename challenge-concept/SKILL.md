## When to use

- You have a rough animation idea and want to validate it before committing time
- You need to decide between output formats (prototype, GIF, short movie) and aren't sure which fits
- You want to uncover technical risks or emotional gaps in your motion concept early

## How it works

This is a structured conversation. No files are produced — the output is clarity.

### Rules

**ONE question per message. ALWAYS with lettered options. NO exceptions.**

- Ask a single question. Wait for the answer. Then ask the next one.
- Every question MUST have A/B/C/D options — never ask as open-ended text.
- If you catch yourself writing more than one question mark in a message, STOP and split them.
- Recommend one option: "I'd suggest B because..." — let the user agree or pick differently.
- Skip questions the codebase or previous answers already resolve.

### Start here

Begin with this single question:

**What are you animating?**

Wait for the answer. Then ask the next question based on what they said. Follow this general order, but skip or adapt based on context:

### Question flow

Ask these one at a time. Each one is a separate message. For multiple-choice questions, display the options exactly as shown — do not convert them to open-ended questions.

**Question 1** (open):
Ask: "What are you animating?"

**Question 2** (open):
Ask: "Do you have a Figma file? Paste the link."

**Question 3** (display these choices):
Ask: "What's the focus area for the animation?"
- A) The entire screen — full page animation
- B) A section of the page — like a hero area or content block
- C) A floating element — like a tray, modal, popover, or tooltip
- D) A single component — like a button, card, or input field
- E) A transition between screens — navigating from one view to another

**Question 4** (display these choices):
Ask: "What's the output format?"
- A) Prototype — interactive, code-based
- B) GIF — exported animation file
- C) Short movie / video
- D) Not sure yet — help me decide

**Question 5** (display these choices, allow multiple):
Ask: "Where will people see it? You can pick more than one."
- A) Website / web app
- B) Mobile app
- C) Social media
- D) Presentation
- E) Other

**Question 6** (display these choices):
Ask: "What should the viewer feel?"
- A) Excited / energized
- B) Calm / reassured
- C) Impressed / wowed
- D) Focused / guided
- E) Playful / amused
- F) Something else — describe it

**Question 7** (display these choices):
Ask: "How fast should the animation feel?"
- A) Instant
- B) Quick
- C) Moderate
- D) Slow
- E) Cinematic

**Question 8** (display these choices):
Ask: "How long is the animation?"
- A) Under 1 second
- B) 1–3 seconds
- C) 3–10 seconds
- D) 10+ seconds
- E) Not sure

**Question 9** (display these choices):
Ask: "Does it loop?"
- A) No — plays once
- B) Yes — seamless loop
- C) Yes — with a pause between loops
- D) Not sure

**Question 10** (display these choices, allow multiple):
Ask: "What assets do you have? Pick all that apply."
- A) Illustrations / icons
- B) Existing code components
- C) Brand guidelines
- D) Nothing beyond the Figma

**Question 11** (open):
Ask: "If you removed this animation entirely, what would the user lose?"

**Question 12** (display these choices):
Ask: "Is this for the user's benefit or mostly aesthetic?"
- A) User benefit — makes something clearer or easier
- B) Both — looks great AND serves a purpose
- C) Mostly aesthetic — and that's fine

### When to stop

Stop when every question is answered and there's no ambiguity left. You don't need to ask all 10 — if earlier answers already cover later questions, skip them.

Summarize what you've learned and confirm: "Here's what I understand about your animation — does this sound right?"
